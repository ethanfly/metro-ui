Phase 0 · Philosophy

Four principles from the source

The article traces Metro to the Bauhaus of Walter Gropius and Ludwig Mies van der Rohe ——littleess is more,—proportion over decoration, and detail in the places you don—don't feel. Metro revives that lineage for the screen.

Bauhaus lineage

Not arbitrary stacking. Gropius and Mies van der Rohe treated proportion, light & shadow, typography and image as a carefully matched system —Metro inherits that rigor.

Proportion first

Block volume, line spacing and hierarchy are calibrated in the wireframe —before color. Size itself carries meaning.

Color table first

Establish the full palette up front —dark vs. light, warm vs. cool —then fill the wireframe from it. Backgrounds differentiate, not gradients.

Zero-learning

Interaction must be so intuitive that users don—don't need to learn it. Animation, ergonomics, tactility and sound all serve that goal.

让交互变得更简单易懂却又不简陋。

Make interaction simpler and more understandable —yet never crude.

About

Metro UI

A self-contained design system and showcase, built from the design philosophy described in —Modernetro UI—on Medium —proportion-first wireframes, a fixed color table, light-weight typography, and the Bauhaus lineage of Gropius and Mies van der Rohe.

No frameworks, no build step —plain HTML, CSS custom properties, and a sprinkle of vanilla JS. Open styles/tokens.css to retune the color table; everything else inherits.
